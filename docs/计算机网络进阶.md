# 计算机网络进阶面试题

## OSI 七层模型和 TCP/IP 模型如何对应？

OSI 包括物理、数据链路、网络、传输、会话、表示、应用七层。TCP/IP 实践中常划分为链路层、网络层、传输层和应用层。HTTP、DNS 属于应用层，TCP、UDP 属于传输层，IP 属于网络层。

## TCP 四次挥手为什么通常需要四次？

TCP 是全双工协议，双方的发送方向需要分别关闭。主动关闭方发送 FIN，对方先回复 ACK；对方完成剩余数据发送后再发送 FIN，主动方回复 ACK。ACK 和 FIN 有时可以合并，但语义上仍是两个方向各自关闭。

## TIME_WAIT 状态有什么作用？

主动关闭方在发送最后 ACK 后进入 TIME_WAIT，等待足够时间，确保最后 ACK 丢失时能重发，也避免旧连接中的延迟报文影响相同四元组的新连接。大量 TIME_WAIT 应先分析连接复用和关闭方式，不能简单粗暴取消。

## TCP 如何实现可靠传输？

TCP 使用序列号、确认应答、校验和、超时重传、快速重传和滑动窗口保证数据有序、无重复地交付。接收方通过 ACK 反馈已收到的数据，发送方在丢包或超时后重传。

## 流量控制和拥塞控制有什么区别？

流量控制避免发送方速度超过接收方处理能力，主要依据接收窗口。拥塞控制避免向网络注入过多数据，主要依据拥塞窗口和丢包、时延等信号。实际发送窗口通常取两者较小值。

## DNS 查询过程是什么？

客户端先查浏览器、系统和本地 DNS 缓存。未命中时向递归解析器查询，解析器可能依次访问根服务器、顶级域服务器和权威 DNS 服务器，得到记录后返回并缓存。实际过程会受缓存、CNAME 和 DNS 配置影响。

## HTTP/1.1、HTTP/2 和 HTTP/3 有什么主要区别？

HTTP/1.1 支持持久连接，但同一连接上的请求响应存在队头阻塞限制。HTTP/2 在一个 TCP 连接上进行二进制分帧和多路复用，并支持头部压缩。HTTP/3 基于 QUIC，在 UDP 上实现传输，减少连接建立成本并缓解传输层队头阻塞。

## HTTP 常见状态码有哪些？

200 表示成功，201 表示已创建，204 表示成功但无响应体；301 和 308 是永久重定向，302 和 307 常用于临时重定向；400 是请求错误，401 是未认证，403 是无权限，404 是资源不存在，429 是请求过多；500、502、503、504 表示服务端或网关问题。

## Cookie、Session 和 Token 有什么区别？

Cookie 是浏览器保存并按规则随请求发送的小段数据。Session 通常把会话状态保存在服务端，客户端只持有会话标识。Token 把认证凭证交给客户端保存，服务端验证其签名或查询状态。安全设计还需考虑 HTTPS、过期、刷新、撤销和 CSRF。

## 什么是跨域和 CORS？

浏览器同源策略限制不同协议、域名或端口之间的脚本访问。CORS 由服务端通过响应头声明允许的来源、方法和请求头。复杂请求会先发送 OPTIONS 预检。CORS 是浏览器安全机制，不是服务器之间请求的限制。

## 正向代理和反向代理有什么区别？

正向代理代表客户端访问外部服务，服务端通常只看到代理地址；反向代理代表服务端接收客户端请求，再转发到后端实例。反向代理常用于负载均衡、TLS 终止、缓存、限流和隐藏内部拓扑。

## WebSocket 与 HTTP 有什么关系？

WebSocket 通常先通过 HTTP Upgrade 握手建立连接，之后在同一 TCP 连接上进行全双工消息传输。它适合聊天室、协同编辑和实时推送。心跳、断线重连、消息顺序和扩容时的连接路由需要由应用设计。
