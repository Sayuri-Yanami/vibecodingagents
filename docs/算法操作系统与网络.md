# 算法、操作系统与网络

## 动态规划和贪心算法有什么区别？

动态规划把问题拆为相互重叠的子问题，保存子问题结果，并依赖最优子结构得到全局最优解。贪心算法在每一步选择当前看来最优的方案，只有在问题满足贪心选择性质时才能保证全局最优。设计动态规划时通常明确状态、转移方程、初始条件和遍历顺序。

## 二分查找有哪些前提和常见错误？

二分查找通常要求搜索空间具有单调性，最常见场景是有序数组。常见错误包括左右边界定义不一致、循环条件与更新方式不匹配、整数溢出，以及查找第一个或最后一个满足条件的位置时没有保存候选答案。

## 进程和线程有什么区别？

进程是操作系统进行资源分配和隔离的基本单位，通常拥有独立虚拟地址空间。线程是 CPU 调度的基本单位，同一进程内的线程共享代码段、堆和打开的文件等资源，但各自拥有栈、寄存器和程序计数器。线程切换通常比进程切换轻量，但共享数据也带来同步风险。

## 死锁产生的四个必要条件是什么？

四个必要条件是互斥、请求并保持、不可剥夺和循环等待。预防死锁可以破坏其中至少一个条件，例如统一资源申请顺序以破坏循环等待；避免死锁还可以通过银行家算法等方式只批准安全状态下的资源请求。

## TCP 为什么需要三次握手？

第一次握手由客户端发送 SYN，表达建立连接的请求和客户端初始序列号。第二次由服务器回复 SYN+ACK，确认客户端请求并发送服务器初始序列号。第三次由客户端发送 ACK，使服务器确认客户端能够收到服务器的响应。三次握手能确认双方的发送与接收能力，并同步双方初始序列号；两次握手不足以让服务器确认自己的响应已被客户端接收。

## TCP 和 UDP 的主要区别是什么？

TCP 面向连接，提供可靠、有序、无重复的字节流，并具有流量控制和拥塞控制。UDP 无连接，以数据报为单位，不保证可靠到达和顺序，协议开销较小、时延更低。实时音视频、DNS 等场景常使用 UDP，并在应用层按需要补充可靠性机制。

## HTTP 和 HTTPS 有什么区别？

HTTPS 是运行在 TLS 之上的 HTTP。TLS 提供服务器身份认证、传输加密和完整性校验，降低窃听与篡改风险。建立 HTTPS 连接时通常先完成 TCP 连接，再进行 TLS 握手；使用 HTTP/3 时底层改为基于 UDP 的 QUIC。
