# Python 面试题

## 装饰器是什么？

装饰器本质上是一个接收函数或类并返回新对象的可调用对象。它可以在不修改原函数源码的情况下增加日志、权限校验、缓存、计时等横切能力。实现函数装饰器时通常使用闭包，并用 `functools.wraps` 保留原函数的名称、文档字符串等元数据。

## 生成器和普通函数有什么区别？

普通函数执行到 `return` 后结束并一次性返回结果。生成器函数包含 `yield`，调用时返回生成器对象；每次迭代执行到 `yield` 暂停，并保存当前栈帧状态，下次从暂停位置继续。生成器按需产生数据，适合处理大文件、数据流和无限序列，能显著降低内存占用。

## Python 的 GIL 是什么？

CPython 的全局解释器锁（GIL）保证同一进程中同一时刻通常只有一个线程执行 Python 字节码。它简化了 CPython 的内存管理，但会限制 CPU 密集型多线程的并行能力。I/O 密集型任务仍可受益于多线程；CPU 密集型任务通常考虑多进程、原生扩展或其他运行时。

## 深拷贝和浅拷贝有什么区别？

浅拷贝创建一个新容器，但容器内部仍引用原来的子对象，因此修改可变子对象可能同时影响原对象和副本。深拷贝会递归复制子对象，使两份对象尽量独立。Python 中可分别使用 `copy.copy` 和 `copy.deepcopy`。

## async 和 await 的作用是什么？

`async def` 定义协程函数，调用后得到协程对象。`await` 用于挂起当前协程，等待另一个可等待对象完成，同时把事件循环的执行权交给其他任务。异步编程适合大量网络、磁盘等 I/O 等待场景，不等同于自动获得 CPU 并行能力。
