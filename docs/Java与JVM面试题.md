# Java 与 JVM 面试题

## Java 内存模型解决什么问题？

Java 内存模型（JMM）定义多线程程序中共享变量的可见性、有序性和原子性规则，并通过 happens-before 关系说明一个操作的结果何时对另一个操作可见。`volatile`、锁、线程启动和线程终止等都能建立特定的 happens-before 关系。

## HashMap 的工作原理是什么？

HashMap 根据键的哈希值定位数组桶。Java 8 中，桶内发生冲突时先使用链表；当链表长度达到阈值且数组容量满足条件时可树化为红黑树，以改善严重冲突下的查询性能。HashMap 默认不保证线程安全，键和值都可以为 null。

## synchronized 和 ReentrantLock 有什么区别？

二者都可实现互斥和可见性。`synchronized` 由 JVM 管理，离开同步块会自动释放锁，语法更简洁。`ReentrantLock` 是显式锁，支持可中断获取、超时获取、公平锁以及多个 `Condition`，但必须在 `finally` 中主动释放。

## JVM 垃圾回收如何判断对象是否存活？

主流 JVM 使用可达性分析：从 GC Roots 出发沿引用关系搜索，无法到达的对象可以被回收。常见 GC Roots 包括线程栈中的局部变量引用、类静态字段引用、JNI 引用等。引用计数不能可靠处理循环引用，因此不是 JVM 的主要判活方法。

## 类加载过程包含哪些阶段？

典型过程包括加载、验证、准备、解析、初始化。加载阶段获得类的二进制数据并创建 Class 对象；验证确保格式和语义安全；准备为静态变量分配内存并设置初始值；解析将符号引用转换为直接引用；初始化执行类初始化方法。
