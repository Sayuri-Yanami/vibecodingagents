# C++ 内存与性能面试题

## 栈和堆有什么区别？

栈通常由编译器和运行时自动管理，用于保存局部变量、函数参数和调用信息，分配释放速度快但空间有限。堆用于动态分配，生命周期由程序控制或智能指针管理，容量更大但分配成本更高，也更容易出现泄漏、悬空指针和碎片。

## RAII 是什么？

RAII 将资源的生命周期绑定到对象生命周期：构造函数获取资源，析构函数释放资源。无论函数正常返回还是抛出异常，栈对象析构时都会完成清理。文件、锁、内存和网络连接都适合使用 RAII 管理。

## unique_ptr、shared_ptr 和 weak_ptr 有什么区别？

`unique_ptr` 表示独占所有权，不能复制但可以移动，开销较小。`shared_ptr` 通过引用计数共享所有权，最后一个持有者销毁时释放对象。`weak_ptr` 不增加强引用计数，用于观察共享对象并打破循环引用，使用前需要通过 `lock()` 检查对象是否仍存在。

## 什么是移动语义？

移动语义允许对象把内部资源所有权转交给另一个对象，而不是执行昂贵的深拷贝。右值引用写作 `T&&`，移动构造和移动赋值通常把源对象置于合法但未指定状态。`std::move` 本身不移动数据，只把表达式转换为可被移动操作接收的右值。

## 什么是悬空指针？

悬空指针指向的对象已经被销毁或释放，继续解引用属于未定义行为。常见来源包括返回局部变量地址、释放内存后继续使用、容器扩容后保留旧迭代器。可通过明确所有权、使用智能指针和遵守容器失效规则降低风险。

## new/delete 和 malloc/free 有什么区别？

`new` 会分配内存并调用构造函数，`delete` 会调用析构函数并释放内存；`malloc` 和 `free` 只处理原始内存，不了解 C++ 对象生命周期。两组接口不能混用。现代 C++ 更推荐容器、智能指针和工厂函数，尽量避免裸 `new`。

## 什么是虚函数和虚函数表？

虚函数支持运行时多态。对象通常保存指向虚函数表的指针，调用虚函数时根据对象的动态类型选择具体实现。基类通过指针释放派生类对象时，基类析构函数应声明为虚函数，否则派生类析构可能不会执行。

## 什么是对象切片？

当派生类对象按值赋给基类对象时，派生类独有部分会被丢弃，只保留基类子对象，这称为对象切片。需要多态时应使用基类引用或指针，而不是按值传递基类对象。

## const、constexpr 和 consteval 有什么区别？

`const` 表示对象不可通过该名字修改，但值不一定在编译期可知。`constexpr` 表示对象或函数可以参与常量表达式，在满足条件时于编译期求值。`consteval` 函数必须在编译期求值，适合强制执行编译期计算。

## vector 扩容时会发生什么？

当容量不足时，`vector` 会申请更大的连续内存，把现有元素移动或复制到新区域，再释放旧内存。扩容会使指向原元素的指针、引用和迭代器失效。已知元素规模时调用 `reserve` 可以减少重复扩容成本。

## 内存对齐有什么作用？

处理器通常按特定边界访问数据更高效，某些架构还要求自然对齐。编译器会在结构体成员之间加入填充字节，因此成员顺序会影响结构体大小。优化布局时可以把对齐要求较高或较大的成员放在前面，但不能牺牲可读性和接口兼容性。

## 如何定位 C++ 内存泄漏和越界问题？

先使用 AddressSanitizer、LeakSanitizer、Valgrind 等工具捕获越界、释放后使用和泄漏，再结合调用栈定位所有权错误。代码层面优先使用 RAII、标准容器和智能指针，并为资源类正确实现或禁用复制与移动操作。
