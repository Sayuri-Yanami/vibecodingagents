# Git 与 Linux 工程实践面试题

## Git 工作区、暂存区和版本库是什么？

工作区是当前可编辑文件，暂存区保存下一次提交准备包含的快照，版本库保存提交对象和分支引用。`git add` 把内容加入暂存区，`git commit` 根据暂存区创建提交。

## git fetch 和 git pull 有什么区别？

`git fetch` 只获取远程引用和对象，不修改当前分支。`git pull` 通常等于先 fetch，再把远程分支 merge 或 rebase 到当前分支。为了先检查差异和控制整合方式，协作中常先使用 fetch。

## 如何安全地解决 Git 冲突？

先确认冲突涉及的两个版本和预期行为，再逐个编辑冲突文件，运行测试后使用 `git add` 标记解决，最后继续 merge、rebase 或 cherry-pick。不要只删除冲突标记而忽略两侧语义，也不要覆盖他人未理解的修改。

## git cherry-pick 适合什么场景？

它把指定提交的改动重新应用到当前分支，适合把独立修复移植到发布分支。大量 cherry-pick 容易产生重复提交和后续冲突，因此长期分支同步通常更适合 merge 或 rebase。

## 如何修改最近一次尚未推送的提交？

可以先调整文件并重新暂存，再使用 `git commit --amend` 修改提交内容或消息。该操作会生成新的提交哈希，因此不应随意修改已被他人基于其开发的共享提交。

## 什么是 detached HEAD？

正常情况下 HEAD 指向分支，分支再指向提交。detached HEAD 时 HEAD 直接指向某个提交，新提交不会自动属于普通分支。若要保留工作，应及时创建新分支。

## Linux 文件权限如何表示？

权限分为所有者、所属组和其他用户三组，每组包含读、写、执行。数字表示中读为 4、写为 2、执行为 1，例如 755 表示所有者可读写执行，其他用户可读执行。目录的执行权限表示可以进入或访问其中路径。

## 软链接和硬链接有什么区别？

硬链接是同一文件 inode 的另一个目录项，通常不能跨文件系统，也通常不能指向目录。软链接是保存目标路径的特殊文件，可以跨文件系统，但目标删除后链接会失效。

## 如何查看 Linux 进程和资源占用？

可用 `ps` 查看进程快照，`top` 或 `htop` 动态观察 CPU 和内存，`free` 查看内存，`df` 和 `du` 查看磁盘，`ss` 查看网络连接，`lsof` 查看进程打开的文件。定位问题时要结合一段时间的趋势，而非单次读数。

## 管道和重定向有什么区别？

管道 `|` 把前一个命令的标准输出作为后一个命令的标准输入。`>` 覆盖写入文件，`>>` 追加写入，`2>` 重定向标准错误。组合命令前应明确每个程序读写的流，避免误覆盖文件。

## 什么是环境变量？

环境变量是进程启动时继承的键值配置，常用于路径、运行模式和密钥位置。子进程通常继承父进程环境，但修改不会反向影响父进程。敏感值不应写入代码、提交到仓库或打印到日志。

## Docker 容器和虚拟机有什么区别？

容器共享宿主机内核，通过命名空间和控制组实现隔离，启动快且镜像较小。虚拟机包含完整客户操作系统，隔离边界更强但资源开销更高。容器不是绝对安全边界，仍需最小权限、镜像治理和运行时防护。
