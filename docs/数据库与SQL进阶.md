# 数据库与 SQL 进阶面试题

## 聚簇索引和非聚簇索引有什么区别？

聚簇索引的叶子节点直接组织或保存整行数据，因此表数据的物理或逻辑顺序与索引键相关，一张表通常只有一个。非聚簇索引叶子节点保存主键或行定位信息，查询非覆盖列时可能需要回表。

## 什么是联合索引的最左前缀原则？

联合索引按多个列依次排序，通常可以支持从最左列开始的连续条件。例如索引 `(a,b,c)` 可支持只用 `a`、使用 `a,b` 或 `a,b,c` 的查询。跳过最左列时通常无法有效利用其有序性，但具体情况仍应以执行计划为准。

## 什么是覆盖索引？

如果一条查询所需的过滤列和返回列都能直接从某个索引获得，就不需要回表，这个索引对该查询形成覆盖。覆盖索引可减少随机 I/O，但加入过多列会增大索引体积和写入成本。

## 索引为什么可能失效？

常见原因包括对索引列使用无法优化的函数或表达式、隐式类型转换、前导模糊匹配、选择性过低、联合索引未满足前缀条件，以及优化器判断全表扫描成本更低。应通过执行计划和真实数据验证，而不是只凭规则猜测。

## 数据库隔离级别有哪些？

SQL 标准定义读未提交、读已提交、可重复读和串行化。级别越高，并发异常越少，但并发能力和实现成本可能受影响。不同数据库对快照读、锁定读和幻读的具体实现并不完全一致。

## 脏读、不可重复读和幻读是什么？

脏读是读取到其他事务尚未提交的数据；不可重复读是同一事务内两次读取同一行得到不同值；幻读是按同一条件再次查询时出现或消失了符合条件的行。隔离级别和数据库实现决定这些现象是否可能发生。

## redo log、undo log 和 binlog 各有什么作用？

以 MySQL/InnoDB 为例，redo log 记录页修改，用于崩溃恢复和持久性；undo log 保存旧版本，用于事务回滚和 MVCC；binlog 是服务器层逻辑日志，常用于复制和时间点恢复。三者层级与用途不同。

## 什么是 WAL？

预写日志要求数据页写入持久存储前，相关日志记录先落盘。发生崩溃时可以根据日志重做已提交操作或撤销未提交操作，从而避免每次事务提交都同步写入所有随机数据页。

## INNER JOIN 和 LEFT JOIN 有什么区别？

`INNER JOIN` 只返回两侧匹配的行。`LEFT JOIN` 保留左表全部行，右表无匹配时以 NULL 填充。使用 LEFT JOIN 时，把右表过滤条件错误地放在 WHERE 中可能排除 NULL 行，使结果退化得像 INNER JOIN。

## WHERE 和 HAVING 有什么区别？

`WHERE` 在分组聚合前过滤原始行，通常更有机会使用索引。`HAVING` 在 GROUP BY 和聚合后过滤分组结果，适合对聚合值设置条件。能在 WHERE 完成的过滤通常应尽早执行。

## 窗口函数与 GROUP BY 有什么区别？

GROUP BY 把多行聚合成每组一行。窗口函数在保留原始行的同时，对相关行集合计算排名、累计值、移动平均等结果。常见函数包括 `ROW_NUMBER`、`RANK`、`SUM OVER` 和 `LAG`。

## 如何查询每个部门工资最高的员工？

可以先按部门使用 `RANK()` 或 `DENSE_RANK()` 对工资降序排名，再筛选排名为 1 的行。若最高工资并列，`RANK` 或 `DENSE_RANK` 能保留所有并列员工；使用 `ROW_NUMBER` 只会选择其中一行。

## COUNT(*)、COUNT(1) 和 COUNT(column) 有什么区别？

`COUNT(*)` 统计行数，`COUNT(1)` 在多数现代优化器中通常得到等价计划。`COUNT(column)` 只统计该列非 NULL 的行。性能不能只靠写法判断，应结合数据库版本、索引和执行计划。

## 什么是数据库连接池？

连接池预先创建并复用数据库连接，避免每次请求都进行网络握手和认证。关键参数包括最大连接数、最小空闲数、获取超时、空闲回收和连接有效性检查。池过大可能让数据库产生过多并发和上下文切换。

## 分库分表会带来哪些问题？

它可以分散数据和写入压力，但会引入路由、跨分片查询、分布式事务、全局唯一 ID、扩容迁移和热点分片等复杂性。应先通过索引、缓存、读写分离和数据归档解决问题，再评估是否真的需要分片。
