# 数据库与 Git 面试题

## 数据库事务的 ACID 是什么？

原子性表示事务中的操作要么全部成功，要么全部回滚；一致性表示事务使数据库从一个合法状态转换到另一个合法状态；隔离性控制并发事务之间的影响；持久性表示已提交结果在故障后仍能恢复。日志、锁和 MVCC 等机制共同支持这些性质。

## B+ 树为什么适合数据库索引？

B+ 树具有较高的扇出，使树高较低，能减少磁盘或页读取次数。非叶子节点主要保存键和子节点指针，叶子节点保存记录或记录位置；叶子节点通常通过链表相连，因此既适合等值查询，也适合范围扫描和排序访问。

## 什么是 MVCC？

多版本并发控制（MVCC）通过保存数据的多个版本，让读操作在一定隔离级别下读取一致性快照，从而减少读写之间的阻塞。具体实现因数据库而异，通常涉及事务版本号、可见性规则和旧版本记录。MVCC 并不意味着所有操作都不需要加锁。

## 如何分析一条慢 SQL？

先确认慢查询及其参数和执行频率，再查看执行计划，关注访问类型、预计扫描行数、索引使用、排序、临时表和回表等信息。之后检查过滤条件、连接顺序、索引选择性与覆盖情况，并用真实数据验证优化前后的耗时和资源消耗。

## Git merge 和 rebase 有什么区别？

`merge` 把两个分支的历史通过一次合并提交连接起来，保留真实的分叉结构。`rebase` 把一组提交重新应用到新的基点上，使历史更线性，但会改写提交哈希。已经共享给他人的公共提交不应随意 rebase，以免给协作带来冲突。

## git reset 和 git revert 有什么区别？

`git reset` 移动当前分支指针，并可按模式影响暂存区和工作区，适合处理尚未共享的本地历史。`git revert` 创建一个新的反向提交来抵消指定提交，不改写已有历史，更适合撤销已经推送和共享的提交。
